Estruturas

Sintax

Estrutura nome_da_estrutura
tipo1 item1
tipo1 item2
...
tipo2 item3
tipoN itemN
fim estrutura

Definição

Até agora todos os dados que tínhamos eram armazenados em variáveis simples (carácter, inteiro, real), ou então armazenávamos um conjunto de valores semelhantes em vectores (arrays).
Mas agora vamos aprender um novo tipo de dados estruturados, as estruturas.
Uma estrutura é um conjunto de dados relacionados, provavelmente de tipos diferentes, sendo por isso chamado de estruturas de dados heterogéneas, ao contrário dos arrays que são homogéneos.
Às componentes armazenadas dentro de uma estrutura, dá-se o nome de campo ou de membro da estrutura.

Regras:
Podem-se definir estruturas de estruturas;
As estruturas podem ser recursivas, isto é, podem-se chamar a si mesmas;
Para aceder ao membro yyy da estrutura zzz utiliza-se o operador ponto (.), isto é zzz.yyy.


NOTA: Todas as variáveis da estrutura têm de ser que ser declaradas no momento de definição da estrutura, pois a estrutura deixa de ser identificada fora do local de definição.

Exemplo: Estes exemplos são a declaração de duas estruturas a estrutura pessoa com os campos num. A estrutura turma que tem o campo texto nome e duas estruturas a pessoa e Data.

Exemplos

Estrutura Pessoa
     Inteiro num
     Data dn
Fim estrutura

Estrutura turma
     Texto nome
     Pessoa alunos [20]
     Data exames [2][3]
Fim estrutura
